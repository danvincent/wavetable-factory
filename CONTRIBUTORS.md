# Contributors

This file documents how to contribute changes to the repository.

## Local setup

```bash
npm ci
```

## Build and test

Run checks before opening or updating a PR:

```bash
npm run lint
npm test
```

## Pull request instructions

1. Create a focused branch for one change.
2. Keep commits small and scoped to the issue.
3. Ensure lint and tests pass locally.
4. Open a PR with a clear summary of what changed and why.
5. Address review feedback with follow-up commits.
